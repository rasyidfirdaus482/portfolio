# Building a Modern Portfolio with Next.js

After months of planning and iterating, I finally launched my personal portfolio. In this post, I'll walk you through the technology choices, design decisions, and lessons learned along the way.

## Why Next.js App Router?

The App Router in Next.js 14+ introduces a new paradigm for building React applications. It provides:

- **Server Components** by default — faster initial page loads
- **File-based routing** with layouts and nested routes
- **Static Site Generation (SSG)** for blazing-fast performance
- **Built-in SEO** optimization with the Metadata API

```typescript
// Example: Generating static params for blog posts
export async function generateStaticParams() {
  const posts = getAllPosts('blog');
  return posts.map((post) => ({ slug: post.slug }));
}
```

## The Design Philosophy

I wanted a design that feels **premium** without being over-the-top. The key principles were:

1. **Clean typography** — Using the Geist font family for a modern feel
2. **Subtle animations** — Fade-in effects and smooth transitions
3. **Dark mode first** — Because developers love dark mode
4. **Responsive by default** — Mobile experience is just as important

Here's a visual breakdown of the grid background effect with spotlight that follows the cursor:

The spotlight effect is achieved using CSS `radial-gradient` masks that follow the mouse pointer position, creating an interactive and immersive experience.

## Performance Results

By using static generation and optimized fonts, the portfolio scores **100/100** on Lighthouse for Performance, Accessibility, Best Practices, and SEO.

Key optimizations:
- **Image optimization** with Next.js `<Image />` component
- **Font subsetting** to reduce payload size
- **CSS Modules** to eliminate unused styles
- **Incremental Static Regeneration** for dynamic content
